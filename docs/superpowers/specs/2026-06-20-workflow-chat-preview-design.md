# 工作流测试聊天预览设计

## 目标

点击工作流画布右上角“测试运行”后，在右侧打开与节点详细配置抽屉一致的聊天预览窗口。用户可按照“用户输入”节点配置填写文本或附件，并在当前窗口内连续进行多轮测试对话。

## 界面结构

- 测试预览与节点详细配置共用右侧抽屉位置和视觉样式，两者互斥显示。
- 顶部仅显示标题“预览”和关闭按钮。
- 中间为可滚动消息区，分别展示用户消息、智能体回复、等待状态和错误消息。
- 底部为固定输入区，消息区独立滚动。

## 动态输入

输入区以工作流的“用户输入”节点 `inputFields` 为唯一来源：

- `text`：显示对应文本输入控件；主要文本字段作为消息内容。
- `file`：显示单文件选择控件和已选文件名称。
- `file[]`：显示多文件选择控件和已选文件列表。
- 每个控件显示字段名称和必填状态。
- 必填字段未完成时禁用发送，并显示简短提示。
- 未配置输入字段时，显示一个默认消息文本框，保持工作流可测试。

## 对话行为

1. 点击“测试运行”只打开预览，不立即调用运行接口。
2. 用户填写当前轮输入并点击发送。
3. 发送后立即追加用户消息，并显示智能体等待状态。
4. 复用现有智能体运行接口，成功后追加智能体回复；失败后追加错误消息。
5. 每轮发送后保留消息记录，允许继续输入下一轮。
6. 关闭预览后清空本次测试会话和临时附件。
7. 点击任意节点时关闭测试预览并切换到节点详细配置。

## 数据边界

- 本功能不修改工作流节点配置和自动保存数据。
- 聊天记录只保存在当前预览窗口的前端状态中，不持久化。
- 文件仅作为预览输入状态展示；运行请求沿用当前接口能力，不扩展文件上传后端协议。
- 文本运行请求使用本轮动态文本输入；模型配置和知识库参数继续使用当前工作流配置。

## 异常与空状态

- 没有消息时展示简洁引导语。
- 运行请求失败时在消息流中展示错误气泡，保留之前的对话。
- 运行中禁止重复发送，关闭按钮仍可用。
- 无可用工作流时禁用发送并提示无法运行。

## 验收标准

1. 点击“测试运行”后右侧出现标题为“预览”的聊天窗口，顶部没有其他标题或操作。
2. 输入区根据 `text`、`file`、`file[]` 正确生成控件，并固定在窗口底部。
3. 必填输入未完成时不能发送。
4. 发送后按顺序显示用户消息和智能体回复，支持连续多轮。
5. 关闭预览会清空消息；重新打开时恢复空会话。
6. 点击节点会切换回节点配置窗口。
7. 现有节点配置、自动保存、画布操作和直接运行请求参数不发生回归。

## 测试策略

- 先写组件失败测试，覆盖打开预览、动态输入、必填校验、多轮消息、关闭清空和节点配置切换。
- 使用模拟运行接口验证每轮请求内容和回复顺序。
- 运行工作流页面完整测试、类型检查和生产构建。
- 使用应用内浏览器走通“打开预览 → 输入 → 发送 → 再次发送 → 关闭”的真实交互。
