# 智能体页面实际功能设计

## 背景

当前智能体页面已经具备创建草稿、资产表和试运行反馈的 MVP 雏形，但仍存在两个问题：

- 创建动作使用固定示例数据，不是用户填写后的真实创建。
- 页面混入试运行、Trace 成本、最近运行等内容，职责与运行记录页面重叠。

本次设计将智能体页面收敛为企业后台中的“智能体资产管理与创建入口”，不处理运行调试和 Trace 查看。

## 目标

- 用户可以在页面内填写基础信息并创建智能体。
- 用户可以查看智能体资产表，并选择某个智能体查看配置详情。
- 所有前端展示状态使用中文，不暴露英文枚举给业务用户。
- 智能体页面不再出现试运行、最近运行、Trace 成本等运行诊断内容。

## 非目标

- 不实现智能体试运行。
- 不展示最近运行结果或 Trace 详情。
- 不在本轮实现完整工作流编辑跳转。
- 不新增负责人字段，也不在智能体表格中展示负责人。
- 不实现版本管理、发布历史、权限审批等高级能力。

## 交互设计

### 创建智能体

- 顶部主按钮文案为“创建智能体”。
- 页面展示创建表单，字段为“智能体名称”“应用场景”“模型策略”。
- 表单字段使用可见 label，不依赖 placeholder 说明含义。
- 提交时按钮进入加载态，避免重复提交。
- 创建成功后：
  - 刷新智能体列表。
  - 自动选中新创建的智能体。
  - 在详情区展示中文状态和返回的工作流、知识库、工具信息。
- 创建失败时，在表单附近展示可恢复的错误提示。

### 选择智能体

- 表格每行提供“查看”操作。
- 点击“查看”后，当前智能体详情切换为该行对应智能体。
- 当前选中行需要有可感知的视觉状态，不能只依赖颜色；可使用边框、背景和“当前”文本组合。

### 状态中文化

前端负责将接口枚举映射为中文展示。

| 接口值 | 中文 |
| --- | --- |
| draft | 草稿 |
| ready | 就绪 |
| published | 已发布 |
| blocked | 阻断 |
| passed | 通过 |
| review_required | 需复核 |
| online | 在线 |
| degraded | 异常 |
| offline | 离线 |
| guarded | 受控 |

如果遇到未知状态，展示“未知”，并保留低强调样式。

## 数据设计

### 前端类型

现有 `Agent` 类型继续作为页面数据源。页面不展示 `owner`，但不要求本轮移除接口返回中的 `owner` 字段。

创建请求建议扩展为：

```ts
type CreateAgentPayload = {
  name: string;
  scenario: string;
  modelPolicy: string;
};
```

### 后端接口

`POST /api/agents` 接收：

```json
{
  "name": "售后政策助手",
  "scenario": "售后问答与工单分流",
  "modelPolicy": "gpt-4.1 + fallback"
}
```

返回保持当前 `AgentRead` 结构，但 `modelPolicy` 使用用户提交值。`owner` 可继续由后端默认生成，前端不展示。

### 持久化

如果当前数据库模型未保存 `model_policy`，本轮需要补齐模型字段，确保创建后刷新列表不会丢失模型策略。

## UI/UX 约束

根据 `ui-ux-pro-max` 对企业级 AI Agent 管理后台的建议，本页采用数据密集型后台体验：

- 信息层级优先，避免营销式文案。
- 表单必须使用可见 label。
- 表格需要放在可横向滚动容器中，避免窄屏溢出。
- 异步提交必须有加载、成功或失败反馈。
- 状态文本必须中文可读，颜色只作为辅助提示。
- 交互元素需要有清晰 hover、focus、disabled 状态。
- 动效只用于状态切换和行内容进入，持续时间控制在 150 到 300 毫秒。

## 测试计划

### 前端测试

- 页面展示“创建智能体”，不再展示“创建草稿 Agent”。
- 页面不再展示“试运行”“最新运行”“Trace 成本”等文本。
- 用户填写名称、场景、模型策略并提交后，请求体不包含负责人。
- 创建成功后，列表和当前详情显示新智能体。
- 状态枚举在页面中显示为中文。
- 点击表格中的“查看”后，当前详情切换为对应智能体。

### 后端测试

- `AgentCreate` 支持 `modelPolicy`。
- 创建 Agent 后返回提交的 `modelPolicy`。
- 使用 session factory 持久化后，`list()` 返回的 Agent 仍保留 `modelPolicy`。

## 风险与处理

- 现有工作区已有未提交改动：实施时只修改与智能体页面、相关 API 和测试有关的文件，不回退其他改动。
- 如果数据库迁移机制暂未完善，优先保持测试环境模型可创建；真实迁移可以作为后续单独任务处理。
- 工作流页面与智能体详情的深度跳转暂不在本轮实现，避免扩大范围。
