# 工作流节点配置顶部操作栏设计

## 目标

用户从智能体进入工作流画布并选中任意节点后，右侧节点详细配置面板顶部固定展示一行操作按钮。该行只保留“测试运行”和“发布”，减少无关操作并修复发布按钮右侧的异常符号。

## 界面设计

- 将现有画布右上角操作区移入右侧节点配置面板顶部。
- 顶部操作行仅包含“测试运行”和“发布”两个按钮。
- “测试运行”继续复用现有运行状态、禁用状态和点击处理逻辑。
- “发布”按钮仅显示文字“发布”，不显示下拉符号或其他附加字符。
- 删除“环境变量”和“关闭调试”等多余按钮；现有右侧面板关闭按钮继续保留。
- 操作行固定在面板顶部，配置内容在其下方滚动，保证滚动详细配置时两个主操作仍可见。
- 收紧操作行、节点标题和首个配置卡片之间的垂直间距，使配置内容整体上移约 8–12px；不压缩字段内部的可读间距。

## 行为与数据

- 不修改工作流自动保存机制。
- 不修改节点选择、节点配置、测试运行请求或结果处理逻辑。
- 发布按钮本次只整理布局和文案，不扩展发布业务流程。
- 未选中节点或右侧配置面板关闭时，不在画布上额外显示该操作行。

## 响应式与可访问性

- 桌面端操作行占满右侧面板可用宽度，按钮保持可点击且不与关闭按钮重叠。
- 窄屏底部抽屉形态沿用同一操作行，并允许配置内容独立滚动。
- 保留“测试运行”“发布”“关闭节点配置”的可访问名称与键盘操作能力。

## 验收标准

1. 选中任意节点后，右侧配置面板顶部只出现“测试运行”和“发布”。
2. 画布右上角不再显示原操作按钮组。
3. 页面中不存在“环境变量”“关闭调试”按钮，也不存在发布按钮后的“⌄”等异常符号。
4. 配置卡片相较当前布局更靠近顶部，且不与操作行或关闭按钮重叠。
5. 测试运行仍能触发现有运行流程，运行中状态仍可见。
6. 自动保存、节点配置和面板关闭功能保持正常。

## 测试策略

- 组件测试先断言旧布局不符合新要求，再调整结构和样式使测试通过。
- 运行工作流页面相关测试、类型检查和构建。
- 使用应用内浏览器走通“智能体 → 工作流画布 → 点击节点”的目标路径，检查桌面视口下的按钮、间距、滚动和控制台错误；条件允许时补充窄屏检查。
