# 版本发布计划

## 1. 发布策略

发布计划围绕 MVP 主路径逐步收敛：

创建智能体 -> 编排能力 -> 绑定 MCP/API 工具、数据库和知识库 -> 调试运行 -> 评测 -> 发布 API -> 观测与审计。

产品形态是企业级 AI Agent 工作台，不是单纯聊天窗口。里程碑验收需覆盖任务列表、任务执行区、文件区、日志区、工具面板、Agent 配置区、权限与安全区。

每个里程碑都应有可演示、可验收的闭环结果。M0 验证产品原型和关键信息架构，M1 完成 MVP 闭环，M2 增强企业治理和可运营能力。

## 2. M0 原型验证

### 目标

验证核心对象、页面结构和主流程是否能被目标用户理解，降低后续工程实现返工风险。

### 建议范围

- 工作台信息结构原型。
- 任务列表、任务执行区、文件区、日志区、工具面板、权限与安全区原型。
- 智能体列表、详情、创建草稿流程。
- 工作流编排器原型，包括节点库、画布、节点配置面板和调试结果面板。
- MCP Server、API 工具、数据库连接和工具详情原型。
- 知识库统一管理原型，包括知识库列表、详情、文档管理、向量索引管理、检索测试。
- Trace 详情原型。
- 评测、API 发布和发布检查流程原型。
- 基础角色权限说明和关键操作清单。
- 高风险操作确认规则原型。

### 不包含

- 完整后端接口实现。
- 真实模型调用、真实向量检索和真实 MCP 调用。
- 真实数据库查询、代码执行和文件读写。
- 高级审计、企业 IM 渠道、模板市场。

### 关键验收

- AI 应用开发者可以理解如何创建智能体、配置工作流并查看 Trace。
- 企业员工可以理解如何在工作台发起任务、查看执行进度和日志。
- 运营人员可以理解如何统一维护知识库、文档处理、向量索引、测试集并触发发布前评测。
- 管理员可以理解空间、角色、工具密钥、数据库连接、文件访问、高风险确认和权限边界。
- 原型能完整演示“创建智能体到发布测试环境 API”的主路径。
- 团队确认 P0/P1/P2 优先级和领域模型无重大冲突。

### 主要交付物

- 可点击产品原型或低保真页面流。
- P0 功能清单确认版。
- 用户故事确认版。
- 发布门禁规则草案。
- Agent 配置字段确认版。
- 知识库统一资源中心确认版。
- 高风险操作确认规则草案。

### 风险与退出条件

- 风险：工作流编排能力过大，MVP 工程成本不可控。
- 风险：数据库查询、代码执行、本地文件读写、多 Agent 协作能力边界过宽，影响 MVP 聚焦。
- 风险：评测指标和发布门禁规则不够明确，影响闭环验收。
- 风险：知识库统一管理涉及文档处理和索引状态，需尽早确认最小可用流水线。
- 退出条件：目标用户完成原型走查，主流程无阻断级需求歧义。

## 3. M1 MVP 闭环

### 目标

交付最小可用产品，使团队可以从创建智能体到发布测试环境 API，并能通过任务与运行日志诊断问题。

### P0 范围

#### 工作台

- 平台健康指标总览。
- 待处理事项入口，包括运行异常、待发布草稿、评测失败项、MCP 工具异常。
- 任务列表区，展示当前任务、历史任务、任务状态和执行进度。
- 对话/任务执行区，支持输入任务、查看 Agent 回复和执行过程。
- 文件区、日志区、工具面板、Agent 配置区、权限与安全区的基础入口。

#### 智能体管理

- 智能体列表、创建、编辑、归档。
- 基础信息、模型配置、提示词配置、知识库配置、工具配置、安全配置、发布配置和观测配置。
- 智能体详情，展示配置、版本、绑定知识库、MCP/API 工具、数据库连接、文件能力、运行概况。
- 草稿版本与已发布版本区分。

#### 工作流编排

- 可视化节点画布。
- 支持开始、LLM、条件分支、MCP/API 工具调用、知识库检索、数据库查询、代码执行、文件读写、Agent 协作、结束节点。
- 节点参数配置。
- 保存草稿和运行当前草稿。
- 节点级运行结果和错误信息。
- 高风险节点遵守安全配置和确认规则。

#### MCP / 工具

- 注册 MCP Server。
- 连接检查和工具列表同步。
- 查看工具 schema、输入输出参数和状态。
- 注册 API 工具，配置认证方式、请求/响应 Schema 和权限范围。
- 工具节点可选择授权工具。
- 工具调用日志和异常状态。

#### 知识库

- 知识库作为平台级独立资源统一管理。
- 创建知识库并配置业务分类、负责人、可见范围和状态。
- 至少一种本地文件导入方式。
- 文档处理流水线，包括文件解析、内容清洗、文档切分、向量化、索引构建和入库状态。
- 向量索引管理，包括索引状态、文档数量、分片数量、最后更新时间和手动重建。
- 配置基础分段、Embedding 和语义检索参数。
- 检索测试。
- 一个知识库可被多个 Agent 复用，一个 Agent 可绑定多个知识库。
- 绑定知识库到智能体或工作流节点，并按 Agent 设置 topK、相似度阈值、是否返回来源。
- Agent 回答展示引用来源。

#### 数据库连接

- 数据库连接列表和详情。
- 数据库查询节点可选择授权连接。
- 默认只读查询。
- 查询语句、参数、结果摘要、状态、耗时和错误进入 Trace。

#### 调试与运行日志

- 单轮调试。
- 任务历史和调用记录。
- Run 和 RunTrace 记录。
- Trace 详情展示提示词、检索、模型、MCP/API 工具、数据库查询、代码执行、文件操作、Agent 协作、输出、token、耗时、成本和错误。
- 运行列表与基础筛选。

#### 评测

- 创建测试集和测试用例。
- 关键用例标记。
- 批量运行评测。
- 展示通过率、平均延迟、平均成本和工具调用成功率。
- 发布前门禁阻断关键测试失败版本。

#### 发布渠道

- 发布检查。
- 发布为测试环境 API。
- API 渠道基础配置，包括 API 路径、认证方式、调用方白名单、请求参数 Schema、响应格式 Schema、并发限制和日志保留周期。
- Release 记录。
- 第三方 API 调用继承 Agent 的知识库、工具、数据库、文件和代码执行权限边界。

#### 权限与空间

- 空间与项目边界。
- Owner、Admin、Developer、Operator、Viewer 基础角色。
- 列表、详情和资源选择器按授权范围过滤。
- 高风险操作确认规则，包括代码执行、文件写入、数据库写入和外部 API 写操作。

### 建议验收场景

1. 管理员创建空间和项目，配置一个 MCP Server、一个 API 工具和一个只读数据库连接。
2. 运营创建平台级知识库、导入文档、完成向量化和索引构建，并通过检索测试确认引用来源。
3. 开发者创建智能体草稿，配置模型、提示词、知识库、工具、数据库连接、安全规则和观测配置。
4. 开发者在工作流中添加 LLM、知识库检索、MCP/API 工具、数据库查询和条件分支节点。
5. 企业员工或开发者在工作台发起任务，并查看执行进度。
6. 开发者运行草稿调试，并在 Trace 中看到节点输入、输出、耗时和错误。
7. 运营创建测试集，运行批量评测。
8. 关键用例通过后，运营将智能体发布为测试环境 API。
9. 第三方系统调用 API 后，团队可以在任务与运行日志中查看调用记录和 Trace。
10. 当工具异常、知识库索引异常、数据库权限异常、高风险规则缺失或关键评测失败时，发布检查给出明确阻断原因。

### 成功指标

- 从创建智能体到发布测试环境 API 的路径可闭环。
- P0 功能无阻断级缺口。
- 每次运行可追踪、可诊断、可复现，包括模型、检索、工具、数据库、代码、文件和 Agent 协作步骤。
- MCP 工具、API 工具、数据库连接和知识库作为一等资源被管理和绑定。
- 知识库支持统一管理、文档向量化和索引构建。
- 数据库查询默认按只读查询设计。
- 高风险操作需要安全确认。
- 非技术运营人员可以完成知识库、测试集和发布检查相关配置。

### 不包含

- 完整插件市场。
- 多租户商业计费。
- 自动化智能体生成。
- 复杂 Agent-to-Agent 协作。
- 完整多 Agent 协作治理。M1 只保留能力边界、节点占位和 Trace 记录口径。
- 高级灰度发布和 A/B 实验。
- 完整企业 SSO。
- 企业微信、飞书、钉钉等企业 IM 正式接入。
- 企业网盘、对象存储、Git 仓库、Confluence、飞书/钉钉文档等完整连接器。
- 完整混合检索优化。M1 优先语义检索。

### 风险与缓解

- 风险：真实 MCP Server 连接和 schema 同步复杂度高。
  - 缓解：M1 先支持有限 transport 和清晰的 schema 校验错误。
- 风险：API 工具和数据库查询带来安全风险。
  - 缓解：M1 默认只读数据库查询，并对外部写操作设置高风险确认。
- 风险：知识库导入、向量化和索引构建质量影响用户感知。
  - 缓解：M1 至少保证一种稳定导入方式，并在检索测试中展示命中依据和索引状态。
- 风险：评测门禁定义过宽或过窄。
  - 缓解：M1 优先采用关键用例通过/失败、工具状态、知识库索引状态、数据库权限状态作为硬门禁。
- 风险：权限模型复杂导致开发周期延长。
  - 缓解：M1 先实现空间、项目、基础角色、Agent 授权资源和高风险确认，细粒度资源级权限持续增强。

## 4. M2 企业治理增强

### 目标

在 MVP 闭环稳定后，补齐企业团队在权限、审计、发布风险、运营分析上的增强能力。

### 建议范围

#### 资源级权限

- 智能体级、工具级、API 工具级、数据库连接级、文件访问级、知识库级授权。
- 工具选择器、知识库选择器、数据库连接选择器和发布检查读取资源级权限。

#### 审计日志

- 记录发布、回滚、密钥变更、权限变更、工具配置变更、数据库权限变更、知识库权限变更和高风险操作确认。
- 支持按操作类型、资源、操作者和时间筛选。

#### 发布治理

- 回滚到历史版本。
- 发布检查增强，包括工具状态、知识库索引状态、数据库权限、关键评测结果、高风险确认规则和权限配置。
- 环境级发布策略预留。

#### 运营分析

- 工作台趋势指标。
- 按智能体、渠道、环境查看成功率、延迟、成本趋势。
- 高频错误聚合。
- 知识库使用次数、检索命中率、无结果次数和常见查询问题。
- 第三方 API 调用方分析。

#### 评测增强

- 人工评分。
- 评测报告汇总。
- 指标阈值配置预留。

#### 知识库增强

- 网页或 API 导入。
- 本地目录接入预留。
- 低质量文档和知识空白识别。

#### 渠道增强

- Web Chat 渠道配置完善。
- 企业 IM 渠道作为预留入口，不作为 M2 必须正式接入项。

### 关键验收

- 管理员可以追溯关键高风险操作。
- 资源级权限能影响工具、数据库连接、文件访问、知识库和智能体访问。
- 运营可以通过趋势和评测报告判断版本质量变化。
- 已发布版本出现问题时可以回滚到历史版本。
- 发布检查能呈现可操作的阻断原因和跳转入口。

### 成功指标

- 高风险操作均产生审计记录。
- 发布失败或阻断原因可解释、可追踪。
- 权限变更后，未授权资源不会出现在列表、详情和选择器中。
- 运营能够定位主要失败来源和成本/延迟变化。
- 知识库运营可发现低质量文档或知识空白。

## 5. 后续版本候选方向

- 企业 IM 渠道正式接入，包括企业微信、飞书、钉钉。
- 高级模型路由和成本策略。
- 模板市场和智能体模板化创建。
- 高级灰度发布、A/B 实验和流量分配。
- 完整企业 SSO。
- 自动化智能体生成和复杂 Agent-to-Agent 协作。
- 企业网盘、对象存储、Git 仓库、Confluence、飞书/钉钉文档连接器。
- 关键词检索、混合检索和高级召回优化。
