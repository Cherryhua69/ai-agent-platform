# MVP 功能拆解

## 1. 目标与范围

本文件基于 `mvp-requirements.md`、`information-architecture.md` 和 `domain-model.md`，将 MVP 闭环拆解为可开发、可验收的功能清单。

MVP 主路径：

创建智能体 -> 编排能力 -> 绑定 MCP/API 工具、数据库和知识库 -> 调试运行 -> 评测 -> 发布 API -> 观测与审计。

平台形态是企业级 AI Agent 工作台，不是单纯聊天窗口。功能拆解默认覆盖任务列表、任务执行、文件管理、执行日志、工具配置、Agent 配置、权限与安全等工作台区域。

优先级定义：

- P0：MVP 闭环必需能力，缺失会导致主路径无法验证。
- P1：提升闭环完整度、运营效率和治理可信度，建议在 MVP 迭代内完成。
- P2：企业增强或规模化能力，可在 MVP 后扩展。

## 2. 工作台总览

### P0 功能

#### F-DASH-001 平台健康指标总览

- 用户价值：用户进入系统后快速判断当前空间、项目和环境的运行健康状态。
- 功能范围：展示智能体数量、运行次数、成功率、失败率、平均延迟、Token 成本。
- 关键数据：Agent、Run、RunTrace、Release。
- 验收标准：
  - 支持按空间、项目、环境筛选指标。
  - 指标卡片展示当前值，并能区分暂无数据和数据异常。
  - 用户可以从指标入口跳转到运行日志或智能体列表。

#### F-DASH-002 待处理事项入口

- 用户价值：将发布、评测、工具异常等风险集中暴露。
- 功能范围：展示最近运行异常、待发布草稿、评测失败项、MCP 工具异常。
- 关键数据：AgentVersion、EvaluationRun、Tool、Run。
- 验收标准：
  - 每类待处理事项展示数量、最近更新时间和跳转入口。
  - 工具异常和评测失败可以跳转到对应详情。
  - 无待处理事项时展示空状态。

#### F-DASH-003 任务列表与执行进度

- 用户价值：企业员工可以查看当前任务、历史任务、任务状态和执行进度。
- 功能范围：展示任务标题、关联 Agent、状态、进度、创建人、创建时间、最近更新时间。
- 关键数据：Task、Run、RunTrace。
- 验收标准：
  - 用户可以从任务列表进入任务执行详情。
  - 任务状态变化可以在列表中体现。
  - 任务详情可以查看 Agent 回复和执行过程。

### P1 功能

#### F-DASH-004 趋势与对比

- 用户价值：运营人员能判断近期质量和成本变化。
- 功能范围：按日展示运行量、成功率、延迟、成本趋势。
- 验收标准：
  - 支持最近 7 天和 30 天视图。
  - 支持按渠道或环境过滤。

### P2 功能

#### F-DASH-005 自定义看板

- 用户价值：不同团队可配置关注指标。
- 功能范围：保存个人或空间级看板配置。
- 验收标准：
  - 用户可选择指标组件并保存布局。

## 3. 智能体管理

### P0 功能

#### F-AGENT-001 智能体列表

- 用户价值：开发者和运营可以集中管理智能体资产。
- 功能范围：列表展示名称、状态、负责人、标签、项目、最近运行、最近发布版本。
- 关键数据：Agent、AgentVersion、Run、Release。
- 验收标准：
  - 支持按空间、项目、状态、负责人、标签搜索和筛选。
  - 支持进入详情、编辑、复制、归档入口。
  - 已归档智能体默认不出现在活跃列表中。

#### F-AGENT-002 创建与编辑智能体

- 用户价值：用户无需改代码即可创建可配置的智能体草稿。
- 功能范围：配置基础信息、模型、提示词、知识库、工具、安全、发布和观测配置。
- 关键数据：Agent、AgentVersion。
- 验收标准：
  - 必填字段校验包括 Agent 名称、业务场景、负责人、所属部门或项目、模型配置。
  - 模型配置支持模型供应商、模型名称、temperature、max tokens、超时时间和重试次数。
  - 提示词配置支持系统提示词、角色设定、任务边界、禁止事项和输出格式要求。
  - 工具配置支持 MCP 工具、API 工具、数据库连接、代码执行和本地文件读写能力开关。
  - 安全配置支持高风险操作确认规则、文件访问白名单、数据库只读/可写权限、API 调用权限、敏感数据拦截和调用频率限制。
  - 发布配置支持是否发布为 API、API 路径、认证方式、调用方白名单、请求/响应 Schema、并发限制和日志保留周期。
  - 观测配置支持完整执行链路、模型输入输出、工具调用详情、知识库检索详情和错误告警开关。
  - 创建后生成草稿状态智能体和初始草稿版本。
  - 编辑草稿不会覆盖已发布版本快照。

#### F-AGENT-003 智能体详情

- 用户价值：用户在一个入口查看智能体配置、版本、资源绑定和运行概况。
- 功能范围：展示基础信息、当前草稿版本、当前发布版本、绑定 MCP/API 工具、数据库连接、绑定知识库、最近任务、最近运行和发布记录。
- 验收标准：
  - 能区分草稿版本和已发布版本。
  - 可跳转到工作流编排、运行日志、评测和发布配置。
  - 工具异常、数据库权限异常、知识库索引异常、评测失败、发布阻断在详情页可见。

#### F-AGENT-004 版本记录与回滚入口

- 用户价值：用户可追踪每次发布版本并准备回滚。
- 功能范围：查看 AgentVersion、Release、回滚来源和发布说明。
- 验收标准：
  - 每个版本展示版本号、状态、创建人、创建时间和发布渠道。
  - 已发布版本可以进入只读详情。
  - 回滚操作入口仅对有权限用户可见。

### P1 功能

#### F-AGENT-005 智能体复制

- 用户价值：复用已有配置快速创建新智能体。
- 功能范围：复制基础信息、模型配置、提示词、工作流和资源绑定。
- 验收标准：
  - 复制后生成新的草稿智能体。
  - 复制结果不继承原智能体的运行日志、评测记录和发布记录。

### P2 功能

#### F-AGENT-006 模板化创建

- 用户价值：通过模板降低新智能体创建成本。
- 功能范围：从预设模板或团队模板创建智能体。
- 验收标准：
  - 用户可以选择模板并生成草稿智能体。

## 4. 工作流编排

### P0 功能

#### F-WF-001 可视化节点画布

- 用户价值：开发者可用节点方式定义智能体执行流程。
- 功能范围：支持开始、LLM、条件分支、MCP/API 工具调用、知识库检索、数据库查询、代码执行、文件读写、Agent 协作、结束节点。
- 关键数据：Workflow、WorkflowNode。
- 验收标准：
  - 用户可以新增、删除、连接和移动节点。
  - 节点连接关系保存为工作流草稿。
  - 保存时校验必须有开始节点和结束节点。

#### F-WF-002 节点参数配置

- 用户价值：开发者可以配置每类节点的运行参数。
- 功能范围：配置 LLM 模型、提示词、输入输出变量、条件表达式、工具选择、知识库选择、数据库连接、代码执行参数、文件路径范围、Agent 协作目标和检索参数。
- 验收标准：
  - 不同节点类型展示对应配置项。
  - 必填配置缺失时阻止保存或运行，并给出明确原因。
  - 工具、知识库、数据库连接和文件路径只能选择当前用户或 Agent 有权限访问的资源。
  - 数据库写入、文件写入、代码执行和外部 API 写操作按高风险规则触发确认或阻断。

#### F-WF-003 工作流调试运行

- 用户价值：开发者在发布前验证当前流程是否可运行。
- 功能范围：输入测试问题或变量，运行当前草稿工作流，展示节点级结果。
- 关键数据：Run、RunTrace。
- 验收标准：
  - 调试后展示每个节点的输入、输出、耗时、状态和错误。
  - 失败节点在结果面板中高亮并显示错误原因。
  - 调试运行不会改变已发布版本。

#### F-WF-004 代码/函数节点

- 用户价值：支持轻量数据处理和转换逻辑。
- 功能范围：配置代码片段或函数调用，并在 trace 中展示执行结果，是否启用由 Agent 安全配置控制。
- 验收标准：
  - 支持基础输入输出变量映射。
  - 执行失败时记录错误信息。
  - 未开启代码执行权限时不能运行代码节点。

#### F-WF-005 数据库查询节点

- 用户价值：让 Agent 在授权范围内查询企业业务数据。
- 功能范围：选择数据库连接，配置查询语句、参数映射和结果摘要策略。
- 关键数据：DatabaseConnection、RunTrace。
- 验收标准：
  - 默认只允许只读查询。
  - 可写操作必须显式授权并触发高风险确认。
  - 查询语句、参数、结果摘要、状态、耗时和错误进入 Trace。

#### F-WF-006 文件读写节点

- 用户价值：支持读取本地资料、处理上传文件和生成文件。
- 功能范围：选择文件来源、读取或写入范围、输出文件记录。
- 验收标准：
  - 文件访问必须受文件白名单和 Agent 权限控制。
  - 文件操作记录文件名、路径范围、操作类型、状态和错误。

#### F-WF-007 Agent 协作节点

- 用户价值：预留多 Agent 协作能力，并能追踪委托过程。
- 功能范围：选择可协作 Agent、传入任务、接收返回结果。
- 验收标准：
  - 协作调用必须受权限控制。
  - Trace 中可识别发起 Agent、被调用 Agent、输入、输出、状态和耗时。

#### F-WF-008 人工转接节点

- 用户价值：遇到无法自动处理的问题时转人工处理。
- 功能范围：配置转接条件和转接结果。
- 验收标准：
  - 节点运行结果可在 trace 中识别为人工转接。

### P1 功能

#### F-WF-009 多 Agent 协作配置增强

- 用户价值：在基础 Agent 协作节点之上补充更明确的协作约束。
- 功能范围：协作 Agent 白名单、输入输出契约、失败重试和超时策略。
- 验收标准：
  - 可为单个 Agent 配置可协作对象。
  - 协作失败时有明确错误分类。

### P2 功能

#### F-WF-010 高级流程治理

- 用户价值：支持复杂团队协作和流程质量控制。
- 功能范围：工作流差异对比、节点模板、复杂条件校验。
- 验收标准：
  - 用户可以比较两个版本的节点与连接差异。

## 5. MCP / 工具管理

### P0 功能

#### F-TOOL-001 注册 MCP Server

- 用户价值：管理员或开发者可接入外部工具能力。
- 功能范围：配置名称、transport、endpoint、authType、密钥或环境变量。
- 关键数据：McpServer、Tool。
- 验收标准：
  - 成功保存 MCP Server 后可触发连接检查。
  - 连接失败时展示失败原因并保存为异常状态。
  - 密钥字段不在普通详情中明文展示。

#### F-TOOL-002 同步与查看工具列表

- 用户价值：开发者能确认 MCP Server 暴露的工具及 schema。
- 功能范围：同步工具列表、查看工具名称、描述、inputSchema、outputSchema、状态。
- 验收标准：
  - 同步成功后展示工具列表和 lastSyncedAt。
  - schema 异常时工具状态标记为 schema_error。
  - 工具详情可查看输入输出参数。

#### F-TOOL-003 工作流工具选择

- 用户价值：工作流可以调用已授权 MCP 工具。
- 功能范围：工具节点中选择 MCP Server 和 Tool，并配置参数映射。
- 验收标准：
  - 仅展示用户有权限访问且状态可用的工具。
  - 工具状态异常时禁止发布相关版本或触发发布阻断。

#### F-TOOL-004 工具调用日志

- 用户价值：开发者可排查工具调用失败、延迟和权限问题。
- 功能范围：展示工具调用记录、失败原因、延迟、调用量。
- 关键数据：RunTrace、Tool。
- 验收标准：
  - 可从工具详情查看最近调用记录。
  - 可从一次 RunTrace 跳转到工具详情。

#### F-TOOL-005 API 工具管理

- 用户价值：企业可将非 MCP 的内部或外部接口作为 Agent 可调用工具。
- 功能范围：配置接口地址、请求方法、认证方式、请求参数 Schema、响应 Schema、权限范围和状态。
- 关键数据：ApiTool、RunTrace。
- 验收标准：
  - API 工具可在工作流工具节点中选择。
  - API 调用记录输入、输出、状态、耗时和错误。
  - 外部写操作按高风险规则触发确认或阻断。

### P1 功能

#### F-TOOL-006 工具权限策略

- 用户价值：管理员可控制工具可见性和调用范围。
- 功能范围：按角色、空间、项目、智能体配置 permissionPolicy。
- 验收标准：
  - 未授权用户无法在工作流中选择工具。
  - 权限变更记录审计日志。

### P2 功能

#### F-TOOL-007 工具健康监控

- 用户价值：持续发现不可用或性能退化工具。
- 功能范围：定时健康检查、异常告警、可用率统计。
- 验收标准：
  - 工具从 online 变为 degraded/offline 时生成告警。

## 6. 数据库连接管理

### P0 功能

#### F-DB-001 数据库连接列表与详情

- 用户价值：管理员集中管理 Agent 可访问的数据库资源。
- 功能范围：展示连接名称、类型、环境、负责人、状态、访问模式、最近查询时间。
- 关键数据：DatabaseConnection。
- 验收标准：
  - 未授权用户无法查看数据库连接。
  - 连接详情不明文展示敏感凭据。

#### F-DB-002 数据库连接配置

- 用户价值：管理员可为 Agent 配置企业数据库查询能力。
- 功能范围：创建连接，配置数据库类型、环境、endpoint、认证方式、可见范围、访问模式。
- 验收标准：
  - 默认访问模式为只读。
  - 可写模式必须显式配置，并进入高风险操作范围。
  - 连接变更记录审计日志。

#### F-DB-003 数据库查询日志

- 用户价值：管理员和开发者可追踪数据库查询行为。
- 功能范围：展示查询语句、参数、结果摘要、调用 Agent、调用用户或第三方应用、状态、耗时和错误。
- 验收标准：
  - 可从 RunTrace 跳转到查询日志。
  - 敏感字段按策略脱敏展示。

### P1 功能

#### F-DB-004 数据库权限策略

- 用户价值：管理员可按 Agent、角色和环境控制数据库连接使用范围。
- 功能范围：配置连接可见范围、Agent 授权和查询能力边界。
- 验收标准：
  - 未授权 Agent 不能选择数据库连接。
  - 权限变更记录审计日志。

### P2 功能

#### F-DB-005 数据库查询策略增强

- 用户价值：降低复杂查询和敏感数据风险。
- 功能范围：SQL 模板、查询限流、敏感表字段策略、慢查询告警。
- 验收标准：
  - 管理员可按连接配置查询限制。

## 7. 知识库

### P0 功能

#### F-KB-001 创建知识库

- 用户价值：运营人员可将业务知识作为平台级可检索资源统一管理。
- 功能范围：配置名称、描述、业务分类、负责人、可见范围、状态、Embedding 模型、分段策略、默认检索参数。
- 关键数据：KnowledgeBase。
- 验收标准：
  - 创建后知识库进入可导入文档状态。
  - 必填配置缺失时不能保存。
  - 知识库可按部门、业务线、项目和文档类型分类。
  - 知识库支持启用、停用、归档状态。

#### F-KB-002 文档导入与状态查看

- 用户价值：运营人员可上传文档并看到处理结果。
- 功能范围：上传本地文件、查看文档来源、上传人、更新时间、版本号、解析状态、向量化状态、分片数量。
- 关键数据：Document。
- 验收标准：
  - 支持至少一种文档导入方式完成 MVP 验证。
  - 文档处理失败时展示失败状态和原因。
  - 后续来源扩展包括本地目录、企业网盘、对象存储、Git 仓库、Confluence、飞书/钉钉文档。

#### F-KB-003 文档处理流水线

- 用户价值：管理员和运营可以确认文档是否完成可检索处理。
- 功能范围：文件解析、内容清洗、文档切分、向量化、索引构建、入库。
- 关键数据：Document、VectorIndex。
- 验收标准：
  - 每一步展示处理状态和错误信息。
  - 支持失败重试和重新构建索引。
  - 删除文档后同步清理向量数据。

#### F-KB-004 向量索引管理

- 用户价值：知识库维护独立向量索引，便于排查检索问题。
- 功能范围：查看索引状态、文档数量、分片数量、最后更新时间，支持手动重建索引和增量更新。
- 关键数据：VectorIndex、KnowledgeBase。
- 验收标准：
  - 每个知识库维护独立索引状态。
  - 索引构建失败时展示错误信息。
  - 手动重建索引不会改变知识库权限配置。

#### F-KB-005 检索测试

- 用户价值：用户可验证知识库是否能命中正确内容。
- 功能范围：输入测试问题，查看命中文档、片段、相似度和引用来源。
- 验收标准：
  - 检索结果展示 TopK 命中和引用来源。
  - 引用来源至少包括知识库名称、文件名、段落位置、更新时间。
  - 未命中时展示空结果和当前阈值。

#### F-KB-006 绑定到智能体或节点

- 用户价值：工作流可使用知识库检索节点。
- 功能范围：在智能体资源或工作流知识节点中选择一个或多个知识库，并可按 Agent 设置 topK、相似度阈值、是否返回来源。
- 验收标准：
  - 仅可选择有权限且状态可用的知识库。
  - 调试 trace 中展示检索命中和引用片段。
  - 知识库详情可查看被哪些 Agent 引用。
  - 第三方 API 调用 Agent 时继承该 Agent 的知识库权限。

### P1 功能

#### F-KB-007 知识库权限控制

- 用户价值：企业可以控制哪些用户、部门、角色和 Agent 可访问知识库。
- 功能范围：配置用户、部门、角色、Agent 访问范围。
- 验收标准：
  - Agent 只能检索被授权的知识库。
  - 第三方 API 调用也遵守 Agent 知识库权限。
  - 权限变更记录审计日志。

#### F-KB-008 网页或 API 导入

- 用户价值：扩展知识来源，减少人工上传。
- 功能范围：导入网页 URL 或通过 API 写入数据。
- 验收标准：
  - 导入结果记录 sourceType 和 uri。

### P2 功能

#### F-KB-009 知识质量分析

- 用户价值：发现低质量或无命中文档。
- 功能范围：展示使用次数、被哪些 Agent 引用、检索命中率、无结果次数、常见查询问题、低质量文档或知识空白。
- 验收标准：
  - 可按知识库查看最近命中情况。

## 8. 调试、任务与运行日志

### P0 功能

#### F-RUN-001 单轮调试

- 用户价值：开发者快速验证当前智能体输出。
- 功能范围：输入用户问题和变量，运行草稿版本。
- 验收标准：
  - 调试生成 Run 和 RunTrace。
  - 输出最终回答和运行状态。

#### F-RUN-002 Trace 详情

- 用户价值：用户能定位失败运行来自模型、知识检索、工具调用、数据库查询、代码执行、文件操作、Agent 协作还是权限。
- 功能范围：展示用户输入、变量、提示词、知识检索、模型请求、MCP/API 工具调用、数据库查询、代码执行、文件操作、Agent 协作、最终输出、token、耗时、成本、错误。
- 关键数据：Run、RunTrace。
- 验收标准：
  - Trace 按 stepIndex 展示完整执行链路。
  - 每个步骤展示节点、类型、输入、输出、耗时、token 和错误。
  - 失败运行展示 errorCode 和明确错误归类。

#### F-RUN-003 运行列表与筛选

- 用户价值：开发者和运营可查找运行记录。
- 功能范围：按智能体、版本、状态、时间、渠道筛选。
- 验收标准：
  - 筛选后可进入 Trace 详情。
  - 列表展示状态、延迟、成本、渠道和创建时间。

#### F-RUN-004 任务历史与调用记录

- 用户价值：企业员工和管理员可以追溯历史任务和第三方调用。
- 功能范围：保存任务历史、调用方、请求、响应、状态、耗时、错误和关联 Agent。
- 关键数据：Task、Run。
- 验收标准：
  - 用户可查看自己有权限的历史任务。
  - 第三方 API 调用记录展示调用方应用。
  - 任务历史可进入完整 Trace。

### P1 功能

#### F-RUN-005 多轮对话调试

- 用户价值：验证上下文策略和连续对话效果。
- 功能范围：支持多轮消息输入和 trace 展示。
- 验收标准：
  - 每轮运行可关联同一调试会话。

### P2 功能

#### F-RUN-006 错误分析聚合

- 用户价值：识别高频失败原因。
- 功能范围：按错误类型、节点类型、工具、模型聚合。
- 验收标准：
  - 用户可查看最近错误分布。

## 9. 评测系统

### P0 功能

#### F-EVAL-001 测试集与测试用例

- 用户价值：运营人员可维护发布前验证样本。
- 功能范围：创建测试集，维护测试输入、期望结果、关键用例标记。
- 关键数据：EvaluationDataset。
- 验收标准：

  - 关键用例可参与发布门禁。

- 功能范围：选择智能体版本和测试集，批量运行评测。
- 关键数据：EvaluationRun、Run、RunTrace。
- 验收标准：
  - 评测运行展示状态、通过率、平均延迟、平均成本。
  - 单个失败用例可查看对应 Trace。

- 验收标准：

- 功能范围：为评测用例结果添加人工评分和备注。
- 验收标准：
  - 评测报告可展示人工评分汇总。

### P2 功能

#### F-EVAL-005 指标配置

- 用户价值：不同业务场景可定义不同质量阈值。
- 功能范围：配置准确率、成本、延迟、工具成功率阈值。
- 验收标准：
  - 发布门禁可读取自定义阈值。

## 10. API 发布管理

### P0 功能

#### F-REL-001 发布检查

- 用户价值：发布前集中确认版本、评测、工具和知识库状态。
- 功能范围：检查 AgentVersion、EvaluationRun、Tool、ApiTool、DatabaseConnection、KnowledgeBase、VectorIndex、权限和高风险操作配置。
- 关键数据：AgentVersion、Release、EvaluationRun、Tool、ApiTool、DatabaseConnection、KnowledgeBase。
- 验收标准：
  - 检查通过时允许进入发布确认。
  - 检查失败时 ReleaseStatus 为 blocked 或展示阻断原因。
  - 知识库索引异常、工具异常、数据库权限异常、高风险确认规则缺失时给出明确阻断原因。

#### F-REL-002 发布为 API 到测试环境

- 用户价值：完成 MVP 从创建到测试环境 API 发布的闭环，让第三方系统可调用 Agent 能力。
- 功能范围：将通过检查的智能体版本发布为测试环境 API。
- 验收标准：
  - 发布后生成 Release 记录。
  - 智能体状态更新为 testing 或 published 的约定状态。
  - 可在智能体详情和发布记录中查看发布结果。
  - 第三方 API 调用继承该 Agent 的知识库、工具、数据库、文件和代码执行权限边界。

#### F-REL-003 API 渠道配置

- 用户价值：开发者可以通过 API 调用已发布智能体。
- 功能范围：配置 API 路径、认证方式、调用方白名单、请求参数 Schema、响应格式 Schema、并发限制、日志保留周期、访问密钥。
- 验收标准：
  - 发布记录展示渠道为 API。
  - 访问密钥只在创建或重置时展示。
  - API 调用记录可追踪调用方、请求、响应、状态、耗时和错误。

### P1 功能

#### F-REL-004 Web Chat 渠道配置

- 用户价值：运营可通过网页聊天入口验证真实用户体验。
- 功能范围：配置 Web Chat 渠道、访问范围、版本记录。
- 验收标准：
  - 发布记录展示渠道为 Web Chat。
  - 可查看渠道运行状态。

#### F-REL-005 回滚到历史版本

- 用户价值：线上异常时快速恢复稳定版本。
- 功能范围：选择历史 Release，创建回滚记录。
- 验收标准：
  - 回滚后生成新的 Release 记录并关联 rollbackFromReleaseId。
  - 回滚操作记录审计日志。

### P2 功能

#### F-REL-006 企业 IM 渠道预留

- 用户价值：为企业微信、飞书、钉钉等渠道扩展做准备。
- 功能范围：保留渠道类型、配置入口和状态展示。
- 验收标准：
  - 页面可展示待接入渠道状态，不影响 MVP 发布主路径。

## 11. 权限、空间与治理

### P0 功能

#### F-GOV-001 空间与项目边界

- 用户价值：企业团队可按空间和项目隔离资源。
- 功能范围：用户只能访问授权 Workspace、Project、Agent、Tool、ApiTool、DatabaseConnection、KnowledgeBase、Document。
- 关键数据：Workspace、Project、Agent、McpServer、ApiTool、DatabaseConnection、KnowledgeBase。
- 验收标准：
  - 未授权用户无法查看或操作资源。
  - 所有列表和选择器默认按授权范围过滤。

#### F-GOV-002 基础角色权限

- 用户价值：不同角色有不同操作边界。
- 功能范围：支持 Owner、Admin、Developer、Operator、Viewer。
- 验收标准：
  - Viewer 只能查看授权资源。
  - Developer 可编辑智能体和工作流。
  - Operator 可管理知识库、测试集和发布流程。
  - Admin/Owner 可管理空间、用户、密钥和高风险配置。

### P1 功能

#### F-GOV-003 资源级权限

- 用户价值：精细控制智能体、工具、数据库、文件和知识库访问。
- 功能范围：智能体级、工具级、数据库连接级、文件访问级、知识库级授权。
- 验收标准：
  - 工作流选择资源时应用资源级权限。
  - 权限变更记录审计日志。

#### F-GOV-004 高风险操作确认规则

- 用户价值：降低代码执行、文件写入、数据库写入和外部 API 写操作风险。
- 功能范围：配置按能力类型、Agent、用户角色、环境触发的确认或阻断规则。
- 验收标准：
  - 命中高风险规则时必须确认或阻断。
  - 确认记录进入审计日志。
  - API 调用场景也必须遵守高风险规则。

#### F-GOV-005 关键审计日志

- 用户价值：高风险操作可追溯。
- 功能范围：记录发布、回滚、密钥变更、权限变更、工具配置变更、数据库权限变更、知识库权限变更、高风险操作确认。
- 关键数据：AuditLog。
- 验收标准：
  - 审计日志包含 actorId、action、resourceType、resourceId、metadata、createdAt。
  - 管理员可按操作类型、资源、操作者和时间筛选。

### P2 功能

#### F-GOV-006 企业治理增强

- 用户价值：满足更严格企业安全和合规要求。
- 功能范围：完整 SSO 预留、高级环境策略、密钥轮换提醒。
- 验收标准：
  - 可配置环境级发布策略。
