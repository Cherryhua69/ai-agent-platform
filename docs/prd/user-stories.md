# 用户故事

## 1. 角色定义

### AI 应用开发者

负责智能体逻辑、模型配置、工作流编排、MCP/API 工具调用、数据库查询、代码执行、文件读写能力配置和调试排障。关注点是构建效率、运行可诊断性、工具可用性和版本可控。

### AI 产品/运营人员

负责话术配置、知识库统一维护、文档处理、测试集管理、API 发布渠道和效果分析。关注点是业务效果、发布质量、内容维护效率和用户反馈闭环。

### 企业管理员

负责空间、项目、角色权限、工具密钥、数据库连接、文件访问策略、高风险操作确认、环境治理和审计。关注点是资源隔离、权限边界、风险可控和操作可追溯。

## 2. AI 应用开发者用户故事

### US-DEV-001 创建智能体草稿

- 优先级：P0
- 作为：AI 应用开发者
- 我希望：创建一个智能体草稿并配置基础模型、系统提示词和变量
- 以便：开始构建可调试的 AI 应用能力
- 前置条件：开发者拥有目标项目的编辑权限
- 验收标准：
  - 创建时可以填写 Agent 名称、描述、业务场景、所属部门、负责人和所属项目。
  - 可以配置模型供应商、模型名称、temperature、max tokens、超时时间和重试次数。
  - 可以配置系统提示词、角色设定、任务边界、禁止事项和输出格式要求。
  - 保存后智能体状态为草稿，并生成初始草稿版本。

### US-DEV-002 编排基础工作流

- 优先级：P0
- 作为：AI 应用开发者
- 我希望：在画布中编排 LLM、条件分支、知识库检索和工具调用节点
- 以便：定义智能体的执行流程
- 前置条件：已有智能体草稿
- 验收标准：
  - 可以添加、连接、移动和删除节点。
  - 每类节点可以配置对应参数。
  - 缺少开始节点、结束节点或必填参数时无法成功运行。
  - 数据库查询、代码执行、文件读写和 Agent 协作节点受权限和安全配置控制。

### US-DEV-003 选择 MCP 工具

- 优先级：P0
- 作为：AI 应用开发者
- 我希望：在工具调用节点中选择已授权 MCP 工具并配置参数映射
- 以便：让智能体调用外部业务能力
- 前置条件：管理员或开发者已注册 MCP Server 并同步工具
- 验收标准：
  - 工具选择器只展示我有权限访问的工具。
  - 可以查看工具名称、描述、输入输出 schema。
  - 工具不可用或 schema 异常时，节点给出明确提示。

### US-DEV-004 配置 API 工具

- 优先级：P0
- 作为：AI 应用开发者
- 我希望：将内部或外部 API 配置为 Agent 可调用工具
- 以便：让 Agent 调用业务系统接口
- 前置条件：开发者拥有工具配置权限
- 验收标准：
  - 可以配置接口地址、请求方法、认证方式、请求参数 Schema 和响应 Schema。
  - API 工具可在工作流工具节点中选择。
  - API 调用的输入、输出、状态、耗时和错误可在 Trace 中查看。
  - 外部写操作命中高风险规则时触发确认或阻断。

### US-DEV-005 运行草稿调试

- 优先级：P0
- 作为：AI 应用开发者
- 我希望：输入测试问题并运行当前草稿工作流
- 以便：确认流程能正常执行
- 前置条件：工作流已保存且配置完整
- 验收标准：
  - 调试运行生成 Run 和 RunTrace。
  - 结果展示最终输出、运行状态、延迟、token 和成本。
  - 调试运行不影响已发布版本。

### US-DEV-006 查看节点级 Trace

- 优先级：P0
- 作为：AI 应用开发者
- 我希望：查看每个节点的输入、输出、耗时和错误
- 以便：定位失败来自模型、检索、工具、数据库查询、代码执行、文件操作、Agent 协作还是权限
- 前置条件：已有调试或线上运行记录
- 验收标准：
  - Trace 按执行顺序展示所有步骤。
  - 失败步骤展示错误类型、错误详情和关联节点。
  - 工具调用、知识检索、数据库查询、代码执行、文件操作和 Agent 协作步骤可以跳转或定位到对应资源详情。

### US-DEV-007 配置数据库查询节点

- 优先级：P1
- 作为：AI 应用开发者
- 我希望：在工作流中配置数据库查询节点
- 以便：让 Agent 在授权范围内查询企业业务数据
- 前置条件：管理员已配置数据库连接，开发者拥有使用权限
- 验收标准：
  - 只能选择我或该 Agent 有权限访问的数据库连接。
  - 默认只能配置只读查询。
  - 查询语句、参数、结果摘要、状态、耗时和错误进入 Trace。
  - 可写查询必须显式授权并触发高风险确认。

### US-DEV-008 配置文件读写能力

- 优先级：P1
- 作为：AI 应用开发者
- 我希望：配置 Agent 可读取或写入的文件范围
- 以便：处理本地资料、上传文件和生成文件
- 前置条件：管理员已配置文件访问策略
- 验收标准：
  - 只能访问白名单范围内的文件。
  - 文件写入命中高风险规则时触发确认或阻断。
  - 文件操作记录文件名、操作类型、状态和错误。

### US-DEV-009 查看和复制智能体版本

- 优先级：P1
- 作为：AI 应用开发者
- 我希望：查看历史版本并复制已有智能体
- 以便：复用成熟配置并降低重复搭建成本
- 前置条件：已有智能体和版本记录
- 验收标准：
  - 版本记录展示版本号、状态、创建人、创建时间和发布渠道。
  - 复制后生成新的草稿智能体。
  - 复制结果不继承原智能体运行日志、评测记录和发布记录。

### US-DEV-010 配置代码/函数节点

- 优先级：P1
- 作为：AI 应用开发者
- 我希望：在工作流中添加轻量代码或函数节点
- 以便：处理变量转换、格式化和分支前置逻辑
- 前置条件：工作流编辑权限
- 验收标准：
  - 可以配置输入输出变量映射。
  - 执行结果出现在 Trace 中。
  - 执行失败时记录错误信息。
  - 未开启代码执行权限时不能运行代码节点。

## 3. AI 产品/运营人员用户故事

### US-OPS-001 创建知识库并导入文档

- 优先级：P0
- 作为：AI 产品/运营人员
- 我希望：创建知识库并导入业务文档
- 以便：让智能体回答业务知识问题
- 前置条件：运营人员拥有目标空间或项目的知识库管理权限
- 验收标准：
  - 可以配置知识库名称、描述、业务分类、负责人、可见范围、状态、Embedding 模型、分段策略和检索参数。
  - 知识库作为平台级独立资源创建，不依附于单个 Agent。
  - 可以上传至少一种文档类型完成导入。
  - 文档来源、上传人、更新时间、版本号、处理状态和分片数量可见。

### US-OPS-002 管理文档处理流水线

- 优先级：P0
- 作为：AI 产品/运营人员
- 我希望：查看文件解析、内容清洗、文档切分、向量化、索引构建和入库状态
- 以便：确认文档已变成可检索知识
- 前置条件：知识库已有导入文档
- 验收标准：
  - 每个处理步骤展示状态和错误信息。
  - 处理失败时可以重试。
  - 删除文档后同步清理向量数据。

### US-OPS-003 管理知识库向量索引

- 优先级：P0
- 作为：AI 产品/运营人员
- 我希望：查看和维护知识库独立向量索引
- 以便：排查索引构建和检索问题
- 前置条件：知识库已创建
- 验收标准：
  - 可以查看索引状态、文档数量、分片数量和最后更新时间。
  - 可以手动重建索引。
  - 支持增量更新的状态展示。

### US-OPS-004 测试知识库检索效果

- 优先级：P0
- 作为：AI 产品/运营人员
- 我希望：用测试问题查看知识库命中片段和引用来源
- 以便：确认知识库是否能支持真实问答
- 前置条件：知识库已有处理完成的文档
- 验收标准：
  - 输入问题后展示 TopK 命中片段、相似度和来源。
  - 引用来源至少包括知识库名称、文件名、段落位置和更新时间。
  - 未命中时展示空结果和当前阈值。
  - 检索测试不影响已发布智能体。

### US-OPS-005 绑定知识库到智能体

- 优先级：P0
- 作为：AI 产品/运营人员
- 我希望：将知识库绑定到智能体或工作流知识节点
- 以便：智能体运行时可以检索业务知识
- 前置条件：已有智能体草稿和可用知识库
- 验收标准：
  - 一个 Agent 可以绑定多个知识库。
  - 一个知识库可以被多个 Agent 复用。
  - 只能选择我有权限访问且状态可用的知识库。
  - 可以按 Agent 设置 topK、相似度阈值、是否返回来源和无命中处理策略。
  - 调试 Trace 中可以看到命中片段和引用来源。
  - 知识库详情可以查看被哪些 Agent 引用。

### US-OPS-006 创建测试集和关键用例

- 优先级：P0
- 作为：AI 产品/运营人员
- 我希望：维护测试集、测试用例和关键用例标记
- 以便：发布前验证业务效果
- 前置条件：运营人员拥有评测管理权限
- 验收标准：
  - 可以创建测试集并添加测试输入和期望结果。
  - 可以标记关键用例。
  - 测试集可以绑定到智能体版本。

### US-OPS-007 运行发布前评测

- 优先级：P0
- 作为：AI 产品/运营人员
- 我希望：对待发布版本批量运行测试集
- 以便：判断版本是否达到发布质量
- 前置条件：已有智能体版本和测试集
- 验收标准：
  - 评测结果展示通过率、平均延迟、平均成本和工具调用成功率。
  - 失败用例可以查看对应 Trace。
  - 关键用例未通过时发布门禁阻断发布。

### US-OPS-008 发布为 API 到测试环境

- 优先级：P0
- 作为：AI 产品/运营人员
- 我希望：将通过检查的智能体版本发布为测试环境 API
- 以便：完成 MVP 闭环验证并让第三方系统调用
- 前置条件：版本通过发布检查
- 验收标准：
  - 发布后生成 Release 记录。
  - 可以查看版本、环境、API 路径、认证方式、调用方白名单、发布人和发布时间。
  - 可以配置请求参数 Schema、响应格式 Schema、并发限制和日志保留周期。
  - 发布失败时展示明确失败原因。
  - 第三方 API 调用继承该 Agent 的知识库、工具、数据库、文件和代码执行权限边界。

### US-OPS-009 查看运行效果与异常

- 优先级：P1
- 作为：AI 产品/运营人员
- 我希望：在工作台和运行列表中查看成功率、成本、延迟和失败记录
- 以便：持续优化智能体表现
- 前置条件：已有运行数据
- 验收标准：
  - 可以按智能体、环境、渠道和时间筛选运行记录。
  - 工作台展示评测失败、运行异常和待发布草稿。
  - 可以查看任务历史和第三方 API 调用记录。

### US-OPS-010 人工评分评测结果

- 优先级：P1
- 作为：AI 产品/运营人员
- 我希望：对评测结果进行人工评分和备注
- 以便：补充自动指标无法判断的业务质量
- 前置条件：已有评测运行结果
- 验收标准：
  - 可以对单条评测结果添加评分和备注。
  - 评测报告展示人工评分汇总。

### US-OPS-011 分析知识库使用情况

- 优先级：P2
- 作为：AI 产品/运营人员
- 我希望：查看知识库使用次数、检索命中率、无结果次数和常见查询问题
- 以便：发现低质量文档或知识空白
- 前置条件：已有检索日志
- 验收标准：
  - 可以查看知识库被哪些 Agent 引用。
  - 可以查看常见无结果查询。
  - 可以定位低命中文档或知识空白。

## 4. 企业管理员用户故事

### US-ADMIN-001 管理空间和项目

- 优先级：P0
- 作为：企业管理员
- 我希望：创建和管理空间、项目及环境边界
- 以便：按团队和业务隔离智能体资源
- 前置条件：管理员拥有平台管理权限
- 验收标准：
  - 可以查看授权空间和项目。
  - 列表、详情和资源选择器均按授权空间过滤。
  - 未授权用户无法访问空间内资源。

### US-ADMIN-002 分配基础角色

- 优先级：P0
- 作为：企业管理员
- 我希望：为用户分配 Owner、Admin、Developer、Operator、Viewer 角色
- 以便：控制不同用户的操作范围
- 前置条件：管理员拥有用户与角色管理权限
- 验收标准：
  - Viewer 只能查看资源。
  - Developer 可编辑智能体和工作流。
  - Operator 可管理知识库、测试集和发布流程。
  - Admin/Owner 可管理空间、用户、密钥和高风险配置。

### US-ADMIN-003 注册 MCP Server 和密钥

- 优先级：P0
- 作为：企业管理员
- 我希望：配置 MCP Server 的连接信息、密钥和环境变量
- 以便：让团队安全使用外部工具能力
- 前置条件：管理员拥有工具管理权限
- 验收标准：
  - 可以配置 transport、endpoint、authType、密钥和环境变量。
  - 连接检查失败时记录异常状态和失败原因。
  - 密钥不在普通详情中明文展示。

### US-ADMIN-004 管理数据库连接

- 优先级：P1
- 作为：企业管理员
- 我希望：配置数据库连接、访问范围和只读/可写策略
- 以便：让 Agent 安全查询企业数据
- 前置条件：管理员拥有数据库连接管理权限
- 验收标准：
  - 可以配置数据库类型、环境、endpoint、认证方式、负责人和可见范围。
  - 默认访问模式为只读。
  - 可写模式必须显式配置并进入高风险操作范围。
  - 数据库连接变更记录审计日志。

### US-ADMIN-005 配置工具权限

- 优先级：P1
- 作为：企业管理员
- 我希望：按角色、空间、项目或智能体控制工具可用范围
- 以便：降低高风险工具被误用的风险
- 前置条件：已有 MCP Server 和工具
- 验收标准：
  - 未授权用户无法在工作流中选择对应工具。
  - 权限变更记录审计日志。

### US-ADMIN-006 配置知识库权限

- 优先级：P1
- 作为：企业管理员
- 我希望：配置哪些用户、部门、角色和 Agent 可以访问某个知识库
- 以便：保护企业知识资产
- 前置条件：已有知识库
- 验收标准：
  - Agent 只能检索被授权的知识库。
  - 第三方 API 调用时也遵守 Agent 的知识库权限。
  - 权限变更记录审计日志。

### US-ADMIN-007 配置高风险操作确认规则

- 优先级：P1
- 作为：企业管理员
- 我希望：为代码执行、文件写入、数据库写入和外部 API 写操作配置确认或阻断规则
- 以便：降低企业数据和系统风险
- 前置条件：管理员拥有安全策略配置权限
- 验收标准：
  - 可以按能力类型、Agent、用户角色和环境配置规则。
  - 命中规则时必须确认或被阻断。
  - 确认记录进入审计日志。

### US-ADMIN-008 查看审计日志

- 优先级：P1
- 作为：企业管理员
- 我希望：查看发布、回滚、权限、密钥和工具配置变更记录
- 以便：追溯高风险操作
- 前置条件：系统已记录审计事件
- 验收标准：
  - 审计日志展示操作者、操作、资源类型、资源 ID、时间和元数据。
  - 支持按操作类型、资源、操作者和时间筛选。
  - 高风险操作确认、数据库权限变更和知识库权限变更可追溯。

### US-ADMIN-009 管理发布风险

- 优先级：P1
- 作为：企业管理员
- 我希望：发布前检查工具状态、评测结果和权限风险
- 以便：阻止不合规或不可用版本上线
- 前置条件：已有待发布版本
- 验收标准：
  - 工具异常、知识库索引异常、数据库权限异常、关键评测失败或权限配置不完整时阻断发布。
  - 阻断结果展示具体原因和跳转入口。

### US-ADMIN-010 配置企业治理策略

- 优先级：P2
- 作为：企业管理员
- 我希望：配置更严格的环境策略、SSO 预留和密钥轮换提醒
- 以便：满足企业规模化治理需求
- 前置条件：MVP 闭环已稳定运行
- 验收标准：
  - 可配置环境级发布策略。
  - 可识别需要轮换或过期的密钥。

## 5. 跨角色协作故事

### US-CROSS-001 从创建到发布测试环境闭环

- 优先级：P0
- 作为：AI 应用团队
- 我们希望：开发者、运营和管理员围绕同一个智能体完成创建、编排、调试、评测、发布和观测
- 以便：验证平台 MVP 的核心价值
- 验收标准：
  - 开发者可以创建智能体并完成基础工作流调试。
  - 运营可以统一管理知识库、完成文档向量化和索引构建、绑定测试集并运行评测。
  - 管理员可以保障工具、数据库连接、文件访问、权限、密钥和高风险确认规则可用。
  - 通过评测和发布检查后，智能体可以发布为测试环境 API。
  - 第三方 API 调用后可在任务与运行日志中查看调用记录和 Trace。

### US-CROSS-002 企业员工执行任务并追踪过程
