# MVP 功能需求文档

## 1. 产品定位

本项目是一个面向企业 AI 应用团队、开发者和运营人员的智能体综合管理平台。平台能力覆盖智能体创建、工作流编排、MCP 工具接入、知识库、调试观测、评测和多渠道发布。

第一阶段不追求覆盖所有高级能力，重点建立一条完整闭环：

创建智能体 -> 编排能力 -> 绑定工具和知识库 -> 调试运行 -> 评测 -> 发布 -> 观测。

## 2. 目标用户

- AI 应用开发者：负责智能体逻辑、工具调用、工作流编排和调试。
- AI 产品/运营人员：负责配置话术、知识库、测试集、发布渠道和效果分析。
- 企业管理员：负责空间、权限、工具密钥、审计和环境治理。

## 3. MVP 范围

### 3.1 工作台总览

功能：
- 展示智能体数量、运行次数、成功率、平均延迟、Token 成本、失败率。
- 展示最近运行异常、待发布草稿、评测失败项、MCP 工具异常。
- 支持按空间、项目、环境筛选。

验收标准：
- 用户进入系统后能快速判断当前平台健康状态。
- 可以从总览跳转到具体智能体、运行日志、工具或评测页面。

### 3.2 智能体管理

功能：
- 创建、编辑、复制、归档智能体。
- 配置名称、描述、头像、标签、负责人、所属空间。
- 配置模型、系统提示词、变量、上下文策略。
- 管理状态：草稿、测试中、已发布、异常、已归档。
- 查看发布版本、草稿版本和回滚记录。

验收标准：
- 用户可以创建一个智能体并保存为草稿。
- 用户可以进入智能体详情查看配置、版本、绑定资源和运行概况。

### 3.3 工作流编排

功能：
- 提供可视化节点画布。
- 支持节点类型：开始、LLM、条件分支、工具调用、知识库检索、代码/函数、人工转接、结束。
- 支持节点参数配置面板。
- 支持连接节点、保存草稿、运行当前流程。
- 支持查看节点级运行结果和错误信息。

验收标准：
- 用户可以搭建一个包含 LLM、知识库检索、MCP 工具调用和条件分支的基础工作流。
- 调试后能看到每个节点输入、输出、耗时和错误。

### 3.4 MCP / 工具管理

功能：
- 添加 MCP Server。
- 查看 Server 连接状态、工具列表、schema、输入输出参数。
- 配置工具权限、密钥、环境变量。
- 查看工具调用日志、失败原因、延迟和调用量。
- 标记异常工具，阻止发布风险版本。

验收标准：
- 用户可以注册一个 MCP Server 并查看可用工具。
- 工作流节点可以选择 MCP 工具。
- 工具异常时在智能体详情和发布检查中可见。

### 3.5 知识库

功能：
- 创建知识库。
- 上传文档、导入网页或通过 API 导入数据。
- 配置分段策略、Embedding 模型、检索 TopK、相似度阈值。
- 运行检索测试。
- 绑定知识库到智能体或工作流节点。

验收标准：
- 用户可以创建知识库并导入文档。
- 用户可以用测试问题查看检索结果和引用来源。

### 3.6 调试与运行日志

功能：
- 支持单轮测试和多轮对话测试。
- 展示完整 trace：用户输入、变量、提示词、知识检索、模型请求、工具调用、最终输出。
- 展示 token、耗时、模型、成本、错误栈。
- 支持按智能体、运行状态、时间、渠道筛选。

验收标准：
- 用户可以定位一次失败运行是由模型、知识检索、工具调用还是权限导致。

### 3.7 评测系统

功能：
- 创建测试集和测试用例。
- 批量运行评测。
- 展示通过率、准确率、平均延迟、平均成本、工具调用成功率。
- 支持人工评分。
- 支持发布前检查门禁。

验收标准：
- 用户可以在发布前运行测试集。
- 未通过关键测试时，发布流程给出明确阻断原因。

### 3.8 发布渠道

功能：
- 支持发布到 Web Chat 和 API。
- 预留企业微信、飞书、钉钉等渠道。
- 管理发布配置、访问密钥、域名白名单、版本记录。
- 支持回滚到历史版本。

验收标准：
- 用户可以将通过评测的智能体发布到测试环境或生产环境。
- 发布后可以查看版本、渠道和运行状态。

### 3.9 权限与空间

功能：
- 支持多空间、多项目。
- 支持角色：Owner、Admin、Developer、Operator、Viewer。
- 支持智能体级、工具级、知识库级权限。
- 记录关键审计日志。

验收标准：
- 用户只能访问授权空间和资源。
- 高风险操作可追溯。

## 4. 非 MVP 范围

- 完整插件市场。
- 多租户商业计费。
- 自动化智能体生成。
- 复杂 Agent-to-Agent 协作。
- 高级灰度发布和 A/B 实验。
- 完整企业 SSO。

## 5. 成功指标

- 从创建智能体到发布测试环境的路径可闭环。
- 主要配置无需直接改代码。
- 每次运行可追踪、可诊断、可复现。
- MCP 工具和知识库可以成为一等资源被管理。
