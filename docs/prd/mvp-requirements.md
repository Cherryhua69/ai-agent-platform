# MVP 功能需求文档

## 1. 产品定位

本项目是一个面向企业员工、AI 应用团队、开发者和运营人员的企业级 AI Agent 工作台。平台核心目标是让企业员工能够统一接入企业内部资料、数据库、API 与 MCP 工具，配置不同业务场景的智能体，并可将智能体能力以接口形式暴露给第三方系统调用。

平台能力覆盖智能体创建、工作流编排、MCP 工具接入、API 调用、数据库查询、代码执行、本地文件读写、知识库统一管理、调试观测、评测、API 发布和权限审计。

第一阶段不追求覆盖所有高级能力，重点建立一条完整闭环：

创建智能体 -> 编排能力 -> 绑定工具、数据库和知识库 -> 调试运行 -> 评测 -> 发布 API -> 观测与审计。

产品形态不是单纯聊天窗口，而是企业应用工作台。核心交互区域至少包括任务列表区、对话/任务执行区、文件区、日志区、工具面板、Agent 配置区、权限与安全区。

## 2. 目标用户

- AI 应用开发者：负责智能体逻辑、工具调用、工作流编排和调试。
- AI 产品/运营人员：负责配置话术、知识库、测试集、发布渠道和效果分析。
- 企业管理员：负责空间、权限、工具密钥、审计和环境治理。

## 3. 核心场景

1. 检索本地资料与企业知识库。
2. 配置面向不同业务场景的智能体。
3. 将智能体发布为 API，供第三方系统调用。
4. 通过工作台追踪 Agent 执行过程，包括知识库检索、工具调用、数据库查询、代码执行、文件读写与多 Agent 协作。

## 4. Agent 能力边界

Agent 需要支持以下能力边界，具体是否启用由配置与权限控制决定：

- API 调用。
- 代码执行。
- 本地文件读写。
- 数据库查询，默认优先按只读查询设计。
- 多 Agent 协作。
- MCP 工具调用。
- 知识库检索、文档向量化与索引构建。

高风险能力包括可写数据库操作、外部 API 写操作、代码执行、本地文件写入、敏感文件读取和跨 Agent 委托。高风险能力必须支持配置确认规则、访问控制和审计记录。

## 5. MVP 范围

### 5.1 工作台总览

功能：
- 展示智能体数量、运行次数、成功率、平均延迟、Token 成本、失败率。
- 展示最近运行异常、待发布草稿、评测失败项、MCP 工具异常。
- 展示当前任务、历史任务、任务状态和执行进度。
- 提供任务执行区，用于输入任务、查看 Agent 回复和执行过程。
- 提供文件区、日志区、工具面板、Agent 配置入口、权限与安全入口。
- 支持按空间、项目、环境筛选。

验收标准：
- 用户进入系统后能快速判断当前平台健康状态。
- 可以从总览跳转到具体智能体、运行日志、工具或评测页面。
- 用户可以从任务列表进入历史任务，并查看执行进度和完整日志。

### 5.2 智能体管理

功能：
- 创建、编辑、复制、归档智能体。
- 配置基础信息：Agent 名称、描述、业务场景、所属部门、负责人、状态、版本号。
- 配置模型：模型供应商、模型名称、temperature、max tokens、超时时间、重试次数。
- 配置提示词：系统提示词、角色设定、任务边界、禁止事项、输出格式要求。
- 配置知识库：绑定知识库、检索模式、topK、相似度阈值、是否返回引用来源、无命中处理策略。
- 配置工具：可用 MCP 工具、可用 API 工具、可用数据库连接、是否允许代码执行、是否允许本地文件读写、每个工具的权限范围。
- 配置安全：高风险操作确认规则、文件访问白名单、数据库访问权限、API 调用权限、敏感词或敏感数据拦截、调用频率限制。
- 配置发布：是否发布为 API、API 路径、认证方式、调用方白名单、请求参数 Schema、响应格式 Schema、并发限制、日志保留周期。
- 配置观测：是否记录完整执行链路、模型输入输出、工具调用详情、知识库检索详情和错误告警。
- 管理状态：草稿、测试中、已发布、异常、已归档。
- 查看发布版本、草稿版本和回滚记录。

验收标准：
- 用户可以创建一个智能体并保存为草稿。
- 用户可以进入智能体详情查看配置、版本、绑定资源和运行概况。
- Agent 绑定多个知识库时，运行和 API 调用均只能检索该 Agent 被授权的知识库。
- 禁用代码执行、文件读写、数据库查询或工具调用后，对应能力不能在运行中被调用。

### 5.3 工作流编排

功能：
- 提供可视化节点画布。
- 支持节点类型：开始、LLM、条件分支、工具调用、知识库检索、数据库查询、代码/函数、文件读写、Agent 协作、人工转接、结束。
- 支持节点参数配置面板。
- 支持连接节点、保存草稿、运行当前流程。
- 支持查看节点级运行结果和错误信息。

验收标准：
- 用户可以搭建一个包含 LLM、知识库检索、MCP 工具调用和条件分支的基础工作流。
- 调试后能看到每个节点输入、输出、耗时和错误。
- 数据库查询、代码执行、文件操作和 Agent 协作节点必须遵守 Agent 安全配置和权限边界。

### 5.4 MCP / 工具管理

功能：
- 添加 MCP Server。
- 查看 Server 连接状态、工具列表、schema、输入输出参数。
- 管理 API 工具，包括接口地址、认证方式、请求参数、响应 schema 和权限范围。
- 配置工具权限、密钥、环境变量。
- 查看工具调用日志、失败原因、延迟和调用量。
- 标记异常工具，阻止发布风险版本。

验收标准：
- 用户可以注册一个 MCP Server 并查看可用工具。
- 工作流节点可以选择 MCP 工具。
- 工具异常时在智能体详情和发布检查中可见。
- MCP 工具和 API 工具调用均可追踪输入、输出、状态和耗时。

### 5.5 数据库连接管理

功能：
- 创建和管理数据库连接。
- 配置连接名称、类型、环境、认证信息、负责人和可见范围。
- 配置访问权限，默认优先只读查询。
- 在工作流数据库查询节点中选择授权连接。
- 记录数据库查询语句、参数、结果摘要、状态、耗时和错误。

验收标准：
- Agent 只能使用已授权数据库连接。
- 默认数据库权限为只读；可写权限必须显式配置并触发高风险确认。
- 数据库查询步骤可在任务日志和 Trace 中查看。

### 5.6 知识库管理

功能：
- 知识库作为平台级独立资源统一管理，不作为某个 Agent 的附属配置。
- 支持创建多个知识库，一个知识库可被多个 Agent 复用，一个 Agent 可绑定多个知识库。
- 配置名称、描述、业务分类、负责人、可见范围、状态。
- 支持按部门、业务线、项目、文档类型分类管理。
- 支持启用、停用、归档知识库。
- 上传本地文件，后续可扩展本地目录、企业网盘、对象存储、Git 仓库、Confluence、飞书/钉钉文档等来源。
- 记录文档来源、上传人、更新时间、版本号。
- 支持文件解析、内容清洗、文档切分、向量化、索引构建、入库。
- 每个处理步骤展示处理状态和错误信息，支持失败重试和重新构建索引。
- 每个知识库维护独立向量索引，支持查看索引状态、文档数量、分片数量、最后更新时间。
- 支持手动重建索引、增量更新，删除文档后同步清理向量数据。
- 配置分段策略、Embedding 模型、检索模式、TopK、相似度阈值。
- 运行检索测试。
- 绑定知识库到智能体或工作流节点。
- 知识库详情中查看被哪些 Agent 引用。
- 支持按 Agent 设置检索参数，如 topK、相似度阈值、是否返回来源。
- 记录检索日志，归属于具体任务、Agent 和知识库。
- 查看知识库使用次数、检索命中率、无结果次数和常见查询问题。

验收标准：
- 用户可以创建知识库并导入文档。
- 用户可以用测试问题查看检索结果、相似度和引用来源。
- Agent 回答时应展示引用来源，至少包括知识库名称、文件名、段落位置、更新时间。
- Agent 只能检索被授权的知识库，第三方 API 调用 Agent 时也遵守该 Agent 的知识库权限。
- 权限变更需要记录审计日志。

### 5.7 调试、任务与运行日志

功能：
- 支持任务列表，展示当前任务、历史任务、任务状态和执行进度。
- 支持单轮测试和多轮对话测试。
- 展示完整 trace：用户输入、变量、提示词、知识检索、模型请求、工具调用、数据库查询、代码执行、文件操作、Agent 协作、最终输出。
- 展示 token、耗时、模型、成本、错误栈。
- 支持按智能体、运行状态、时间、渠道筛选。
- 保存任务历史、调用记录和审计日志。

验收标准：
- 用户可以定位一次失败运行是由模型、知识检索、工具调用、数据库查询、代码执行、文件操作、Agent 协作还是权限导致。
- 每一步都可追踪，包括模型调用、知识库检索、工具调用、数据库查询、代码执行、文件操作和多 Agent 协作。

### 5.8 评测系统

功能：
- 创建测试集和测试用例。
- 批量运行评测。
- 展示通过率、准确率、平均延迟、平均成本、工具调用成功率。
- 支持人工评分。
- 支持发布前检查门禁。

验收标准：
- 用户可以在发布前运行测试集。
- 未通过关键测试时，发布流程给出明确阻断原因。

### 5.9 API 发布管理

功能：
- 支持将智能体发布为 API，供第三方系统调用。
- 支持发布到 Web Chat 和 API。
- 预留企业微信、飞书、钉钉等渠道。
- 管理发布配置、访问密钥、调用方白名单、域名白名单、请求参数 Schema、响应格式 Schema、并发限制、日志保留周期和版本记录。
- 支持回滚到历史版本。

验收标准：
- 用户可以将通过评测的智能体发布为测试环境或生产环境 API。
- 发布后可以查看版本、渠道和运行状态。
- 第三方 API 调用 Agent 时，继承该 Agent 的知识库、工具、数据库、文件和代码执行权限边界。
- API 调用记录可追踪调用方、请求、响应、状态、耗时和错误。

### 5.10 权限与空间

功能：
- 支持多空间、多项目。
- 支持角色：Owner、Admin、Developer、Operator、Viewer。
- 支持智能体级、工具级、知识库级权限。
- 支持数据库连接级、文件访问级、API 调用级权限。
- 支持高风险操作确认规则。
- 记录关键审计日志。

验收标准：
- 用户只能访问授权空间和资源。
- 高风险操作可追溯。
- 代码执行、文件写入、数据库写入和外部 API 写操作必须经过权限控制和安全确认。

## 6. 非 MVP 范围

- 完整插件市场。
- 多租户商业计费。
- 自动化智能体生成。
- 复杂 Agent-to-Agent 协作。MVP 可保留 Agent 协作能力边界和日志占位。
- 高级灰度发布和 A/B 实验。
- 完整企业 SSO。
- 企业网盘、对象存储、Git 仓库、Confluence、飞书/钉钉文档等完整连接器。
- 关键词检索或混合检索的完整优化。MVP 优先语义检索。

## 7. 成功指标

- 从创建智能体到发布测试环境的路径可闭环。
- 主要配置无需直接改代码。
- 每次运行可追踪、可诊断、可复现。
- MCP 工具和知识库可以成为一等资源被管理。
- 能够创建、配置、测试、发布智能体。
- 能统一管理知识库，支持知识库检索、文档向量化和索引构建。
- MCP 工具能够被成功调用，并可追踪输入、输出、状态和耗时。
- Agent 能够进行数据库查询，默认按只读查询设计。
- 每一步可追踪，包括模型调用、知识库检索、工具调用、数据库查询、代码执行、文件操作和多 Agent 协作。
- 高风险操作需要安全确认。
- 能保存任务历史、调用记录和审计日志。
